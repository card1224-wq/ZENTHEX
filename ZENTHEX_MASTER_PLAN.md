# Zenthex SaaS Master Plan

이 문서는 현재 Zenthex 설계도와 최근 수정사항을 합친 대표 검토용 최종 마스터 플랜입니다.

## 1. 서비스 정의

Zenthex는 AI 기반 SaaS 플랫폼 회사입니다.

- Zenthex Studio: 프롬프트와 2D 도면을 3D 공간/GLB/JPG 결과로 만드는 AI 스튜디오
- Zenthex Trading: 사용자 설정 기반 자동매매 도구와 Signal Guard
- Customer Center: 계정, 구독, Studio, Trading 문의 접수 및 대표 처리
- CEO Dashboard: 대표 전용 운영, 가입자, 구독, 고객 문의, 출시 검토, 긴급 정지 화면

Zenthex는 설치형 프로그램이 아니라 회원가입, 로그인, 구독, PC/모바일 웹, 클라우드 서버 실행을 기반으로 하는 SaaS입니다.

## 2. 사용자 화면 원칙

홈페이지는 모든 방문자에게 Zenthex를 소개하는 브랜드 화면이어야 합니다. 대표로 로그인해도 첫 화면이 `Zenthex Control` 같은 내부 운영 화면으로 바뀌면 안 됩니다.

비로그인 사용자는 다음 흐름을 봅니다.

- Studio 체험하기
- Trading 구조 보기
- 구독 가격 확인
- 회원가입 / 로그인
- 고객센터

로그인 사용자는 상단 메뉴에서 다음을 봅니다.

- 마이페이지
- 고객센터
- 로그아웃
- 대표 계정인 경우 CEO Dashboard

구독자는 본인이 결제한 상품만 사용할 수 있습니다.

- Studio Pro: Studio 생성과 GLB/JPG 저장
- Trading Pro: Trading 실거래 모드
- Ultimate: Studio와 Trading 모두 사용
- Free: 제한 체험과 구조 확인

## 3. 대표 계정 원칙

대표 이메일은 `7foliath@naver.com`입니다.

대표 계정은 결제 없이 Ultimate 권한을 가집니다. 다만 이메일 인증은 일반 사용자처럼 코드로 진행합니다. 로그인/회원가입 화면에는 대표 계정 문구나 대표 이메일을 노출하지 않습니다.

대표는 다음 기능을 사용할 수 있습니다.

- Studio 전체 사용
- Trading 전체 사용
- 가입자 목록 조회
- 사용자 플랜/역할 변경
- 중복 또는 탈퇴 계정 삭제
- 고객 문의 확인 및 처리
- 출시 전 검토
- 긴급 전체 정지

## 4. 계정과 인증

사용자는 이메일 주소로 로그인합니다. 이메일 로그인은 아이디 찾기와 비밀번호 재설정에 유리합니다.

회원가입 필드:

- 이름
- 이메일
- 비밀번호
- 비밀번호 확인
- 생년월일
- 휴대폰 번호
- 휴대폰 인증코드
- 비밀번호 힌트 질문
- 비밀번호 힌트 답변

현재 테스트 인증코드는 `122492`입니다. 정식 출시 전 실제 이메일 발송기와 SMS 발송기를 연결합니다.

## 5. 데이터 보존 원칙

GitHub는 코드 저장소입니다. 회원, 결제, 구독, 문의, API 키 같은 운영 데이터 저장소가 아닙니다.

GitHub 업로드로 사라지면 안 되는 데이터:

- 회원 계정
- 비밀번호 해시
- 이메일/휴대폰 인증 상태
- 구독 상태
- 결제내역과 영수증
- 고객 문의
- Studio 작업 기록
- Trading 설정
- 암호화된 API 키 데이터

정식 유료 서비스 전에는 `ZENTHEX_DATABASE_URL`로 PostgreSQL 같은 영구 DB를 연결해야 합니다.

## 6. 고객센터

고객센터는 안내문만 있는 페이지가 아니라 문의를 접수하는 운영 기능입니다.

고객 문의 유형:

- 계정 / 로그인
- 구독 / 결제 / 영수증
- Zenthex Studio
- Zenthex Trading / Upbit
- 기타 문의

사용자는 `/customer.html`에서 문의를 남기고, 로그인한 사용자는 본인 문의 내역을 확인합니다. 대표는 `/admin.html`에서 문의 목록을 보고 `접수`, `확인 중`, `답변 완료`, `종료` 상태로 관리하며 답변 또는 내부 메모를 저장합니다.

정식 출시 후에는 문의 접수 시 이메일 알림, 답변 이메일 발송, 첨부파일 업로드, 고객센터 SLA를 추가합니다.

## 7. 비용 단계

현재 검토 단계에서는 비용을 최대한 낮춥니다.

- SQLite: 개발 테스트용
- 무료 PostgreSQL: 초기 검증 가능
- 테스트 인증코드: 무료
- Mock 결제: 무료

정식 유료 서비스 단계에서는 비용이 발생합니다.

- PostgreSQL 유료 플랜
- 서버 호스팅
- 실제 이메일 발송
- SMS 인증
- Object Storage
- Studio AI Provider/API
- GPU Worker
- 결제 수수료
- 모니터링

## 8. 결제와 자동결제

구독은 월 자동결제가 기본입니다.

추천 구조:

- 국내 카드/계좌: Toss Payments 자동결제
- 해외 카드: Stripe Subscriptions

결제 흐름:

1. 사용자가 플랜 선택
2. 결제사 체크아웃 또는 빌링키 발급
3. 첫 결제 성공
4. `subscriptions`에 현재 구독 상태 저장
5. `billing_history`에 결제내역과 영수증 저장
6. 매월 자동결제
7. webhook으로 성공, 실패, 해지, 환불 처리

## 9. Studio 설계

Studio는 프롬프트 또는 2D 도면을 3D 결과로 변환합니다.

핵심 기능:

- 프롬프트 입력
- 2D 도면 업로드
- AI Provider Layer
- GLB 생성
- Three.js 미리보기
- JPG 저장
- 다운로드
- 작업 히스토리
- 사용량 제한
- 실패 시 재시도

체험 정책:

- 비로그인 또는 Free 사용자는 같은 IP 기준 하루 1회 보기 전용 체험
- 체험 사용자는 다운로드, 스크린샷, 출력 제한
- Studio Pro와 Ultimate은 GLB/JPG 저장 가능
- 대표는 제한 없이 사용 가능

## 10. Trading 설계

Trading은 수익 보장 서비스가 아니라 자동매매 도구입니다. 현재 우선 대상은 Upbit KRW 마켓이고, Binance는 다음 확장 대상입니다.

핵심 흐름:

1. 전략 설정
2. 투자금 설정
3. 전체 KRW 코인 스캔
4. 상승 후보 필터링
5. 단기 신호 점수화
6. 리스크 검증
7. 실거래 주문
8. 목표 수익률 도달 시 매도
9. 엔진 종료
10. 모바일 알림

투자금 모드:

- Upbit KRW 전액
- Upbit KRW 비율
- 고정 금액

기존 보유 코인을 자동으로 팔아 다른 코인으로 돌리는 기능은 별도 위험 동의가 필요합니다. 기본 엔진은 Zenthex가 이번 실행에서 산 수량만 매도합니다.

## 11. Trading 리스크 원칙

금지 표현:

- 수익 보장
- 무조건 수익
- 손실 없음
- 대신 투자

필수 고지:

Zenthex Trading은 자동매매 도구이며 투자 자문 또는 수익 보장 서비스가 아닙니다. 모든 투자 판단과 손익 책임은 사용자 본인에게 있습니다.

실거래 조건:

- 로그인
- Trading Pro 또는 Ultimate 구독
- 대표 계정은 결제 없이 가능
- 위험 동의
- Upbit API 키
- Upbit 키 인증 완료
- 자산조회/주문 권한
- 출금 권한 금지
- Zenthex FastAPI 서버 공인 IP 허용

GitHub Pages는 Trading 서버가 아닙니다. Upbit 허용 IP에는 Zenthex FastAPI 서버의 outbound IP를 등록해야 합니다. `ZENTHEX_SERVER_PUBLIC_IP`가 비어 있으면 서버가 `api.ipify.org`로 outbound IP를 조회해 표시합니다.

## 12. 모바일 방향

PC와 모바일 웹/PWA를 모두 지원합니다.

모바일 필수 기능:

- 회원가입
- 이메일/휴대폰 인증
- Studio 결과 확인
- Trading 상태 확인
- 엔진 ON/OFF
- 긴급 정지
- Push 알림
- 고객센터 문의

엔진은 사용자 PC가 아니라 Zenthex 서버에서 실행되어야 합니다.

## 15. 최종 결론

Zenthex는 AI Studio와 자동매매 Trading을 결합한 구독형 SaaS입니다. 현재 방향은 MVP 개발과 대표 검토가 가능한 수준입니다. 다만 실제 유료 고객을 받기 전에는 영구 DB, 자동결제, 실제 이메일/SMS, 법적 문서, Trading 리스크 검증, 고객 문의 운영 체계를 반드시 완료해야 합니다.
