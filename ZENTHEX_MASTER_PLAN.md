# Zenthex SaaS Master Plan

이 문서는 현재 Zenthex 설계도와 추가 수정사항을 합친 대표 검토용 마스터 플랜입니다.

## 1. 회사와 서비스

Zenthex는 AI 기반 SaaS 플랫폼 회사입니다.

핵심 서비스는 두 가지입니다.

- Zenthex Studio: 프롬프트와 2D 도면을 3D 공간/GLB로 변환하는 AI 스튜디오
- Zenthex Trading: 사용자 설정 기반 자동매매 엔진과 Signal Guard

서비스는 설치형 프로그램이 아니라 회원가입, 로그인, 구독, 웹/모바일 사용, 클라우드 서버 실행을 기반으로 하는 SaaS입니다.

## 2. 사용자 화면 원칙

홈페이지는 누구에게나 Zenthex를 소개하는 브랜드 화면입니다.

로그인 전 사용자는 체험 중심으로 봅니다.

- Studio 체험하기
- Trading 구조 보기
- 구독 가격 확인
- 회원가입/로그인

로그인한 구독자는 본인이 구독한 상품을 바로 사용할 수 있어야 합니다.

- Studio Pro: Studio 사용과 GLB 다운로드
- Trading Pro: Trading 실거래 모드
- Ultimate: Studio와 Trading 모두 사용

대표 계정은 결제 없이 전체 권한을 갖지만, 홈페이지가 대표 전용 화면처럼 보이면 안 됩니다. 대표 기능은 CEO Dashboard, 마이페이지, 내부 검토 화면에서만 노출합니다.

## 3. 대표 계정 원칙

대표 이메일은 `7foliath@naver.com`입니다.

대표 계정은 다음 권한을 가집니다.

- 결제 없이 Ultimate 권한
- Studio 전체 사용
- Trading 전체 사용
- 가입자 목록 확인
- 사용자 플랜/역할 변경
- 중복 또는 탈퇴 계정 삭제
- 출시 전 검토
- 긴급 전체 정지

로그인/회원가입 화면에는 대표 이메일이나 대표 계정 안내를 노출하지 않습니다.

## 4. 계정과 인증

사용자는 이메일로 로그인합니다. 이메일 로그인은 아이디 찾기와 비밀번호 재설정에 유리합니다.

회원가입 필드는 다음을 포함합니다.

- 이름
- 이메일
- 비밀번호
- 비밀번호 확인
- 생년월일
- 휴대폰 번호
- 휴대폰 인증코드
- 비밀번호 힌트 질문
- 비밀번호 힌트 답변

현재 테스트 인증코드는 `122492`입니다.

정식 출시 전에는 실제 이메일 발송과 SMS 발송을 연결합니다.

- 이메일: SMTP 또는 메일 발송 서비스
- SMS: Naver Cloud SENS, Aligo, Twilio 등

## 5. 데이터 보존 원칙

GitHub는 코드 저장소입니다. 회원과 결제 데이터 저장소가 아닙니다.

GitHub 업데이트로 절대 사라지면 안 되는 데이터는 다음입니다.

- 회원 계정
- 비밀번호 해시
- 이메일/휴대폰 인증 상태
- 구독 상태
- 결제내역
- 영수증
- Studio 작업 기록
- Trading 설정
- API 키 암호화 데이터

정식 운영 전에는 PostgreSQL 같은 영구 DB를 사용해야 합니다.

개발/검토 단계에서는 SQLite 또는 무료 PostgreSQL을 사용할 수 있습니다. 유료 구독자를 받기 전에는 반드시 영구 DB로 전환합니다.

## 6. 비용 단계

현재 검토 단계에서는 큰 비용 없이 진행합니다.

- SQLite: 무료, 개발 테스트용
- 무료 PostgreSQL: 초기 검토 가능
- 테스트 인증코드: 무료
- Mock 결제: 무료

출시 전 테스트 단계에서 비용이 생길 수 있습니다.

- PostgreSQL 유료 플랜
- 서버 호스팅
- 실제 이메일 발송
- SMS 인증
- Object Storage
- Studio AI Provider/API
- GPU Worker

정식 유료 서비스 단계에서는 운영비가 필수입니다.

- PostgreSQL 백업/유지 비용
- 서버 비용
- AI 생성 비용
- 저장소 비용
- 결제 수수료
- SMS/메일 발송 비용
- 모니터링 비용

대표 검토 화면에서는 무료 검토 단계와 유료 운영 단계가 분리되어 있는지 확인해야 합니다.

## 7. 결제와 자동결제

구독은 월 자동결제가 기본입니다.

추천 구조:

- 국내 카드/계좌: Toss Payments 자동결제
- 해외 카드: Stripe Subscriptions

결제 흐름:

1. 사용자가 플랜 선택
2. 결제사 체크아웃 또는 빌링키 발급
3. 첫 결제 성공
4. `subscriptions`에 현재 구독 상태 저장
5. `billing_history`에 결제내역/영수증 저장
6. 매월 자동결제
7. webhook으로 성공/실패/해지/환불 처리

구독 상태에는 다음이 필요합니다.

- 현재 플랜
- 구독 상태
- 결제사
- 결제사 구독 ID
- 다음 결제일
- 마지막 결제 상태

## 8. Studio 설계

Studio는 프롬프트 또는 2D 도면을 3D 결과로 변환합니다.

주요 기능:

- 프롬프트 입력
- 2D 도면 업로드
- AI Provider Layer
- GLB 생성
- Three.js 미리보기
- 다운로드
- 작업 히스토리
- 사용량 제한
- 실패 시 재시도

체험 정책:

- 비로그인 또는 Free 사용자는 같은 IP 기준 하루 1회 보기 전용 체험
- 체험 사용자는 다운로드, 스크린샷, 출력 제한
- Studio Pro와 Ultimate는 다운로드 가능
- 대표는 제한 없이 사용 가능

## 9. Trading 설계

Trading은 수익 보장 서비스가 아니라 자동매매 도구입니다.

현재 우선 대상은 Upbit KRW 마켓입니다. Binance는 다음 확장 대상입니다.

핵심 흐름:

1. 전략 설정
2. 투자금 설정
3. 전체 KRW 코인 스캔
4. 상승 후보 필터링
5. 단기 신호 점수화
6. 리스크 검증
7. 실거래 주문
8. 목표 수익률 도달 시 매도
9. 엔진 종료
10. 모바일 알림

투자금 모드:

- 업비트 KRW 전액
- 업비트 KRW 비율
- 고정 금액

주의:

- 기존 보유 코인을 자동으로 팔아 다른 코인으로 돌리는 기능은 별도 동의가 필요합니다.
- 기본 엔진은 Zenthex가 이번 실행에서 산 수량만 매도합니다.

## 10. Trading 리스크 원칙

금지 표현:

- 수익 보장
- 무조건 수익
- 손실 없음
- 대신 투자

필수 고지:

Zenthex Trading은 자동매매 도구이며 투자 자문 또는 수익 보장 서비스가 아닙니다. 모든 투자 판단과 손익 책임은 사용자 본인에게 있습니다.

실거래 조건:

- 로그인
- Trading Pro 또는 Ultimate 구독
- 대표 계정은 결제 없이 가능
- 위험 동의
- Upbit API 키
- Upbit 키 인증 완료
- 자산조회/주문 권한
- 출금 권한 금지
- 서버 공인 IP 허용

실거래 화면은 다음 순서로 진행합니다.

1. API 키 입력
2. 키 진단으로 실패 원인 확인
3. 키 인증으로 실거래 가능 상태 확인
4. 위험 동의
5. 실거래 시작
6. 서버가 다시 키를 검증한 뒤 실제 주문 실행

## 11. 모바일 방향

PC와 모바일 웹/PWA를 모두 지원합니다.

모바일에서 필요한 기능:

- 회원가입
- 이메일/휴대폰 인증
- Studio 결과 확인
- Trading 상태 확인
- 엔진 ON/OFF
- 긴급 정지
- Push 알림

엔진은 사용자의 PC나 휴대폰에서 도는 것이 아니라 Zenthex 서버에서 실행되어야 합니다.

## 14. 최종 결론

Zenthex는 AI Studio와 자동매매 Trading을 결합한 구독형 SaaS입니다.

현재 방향은 MVP 개발과 대표 검토가 가능한 수준입니다. 다만 실제 유료 고객을 받기 전에는 영구 DB, 자동결제, 이메일/SMS, 법적 문서, 실거래 리스크 검증이 반드시 완료되어야 합니다.
